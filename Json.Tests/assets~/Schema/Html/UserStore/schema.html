Friflo.Json.Fliox.Hub.Host.Auth.Rights namespace

Right abstract class

Each Role has a set of rights.
Each Right is a rule used to grant or deny a specific database operation or command execution.
The database operation or command execution is granted if any of it rights grant access.
type discriminator right type
"allow" AllowRight
"task" TaskRight
"sendMessage" SendMessageRight
"subscribeMessage" SubscribeMessageRight
"operation" OperationRight
"predicate" PredicateRight
description string | null optional description explaining the Right

AllowRight class extends Right

Allow full access to the given database.
type "allow"right type
database string a specific database: 'test_db', multiple databases by prefix: 'test_*', all databases: '*'

TaskRight class extends Right

TaskRight grant database access by a set of task types.
type "task"right type
database string a specific database: 'test_db', multiple databases by prefix: 'test_*', all databases: '*'
types TaskType[] set fo task types like: create, read, upsert, delete, query, ...

SendMessageRight class extends Right

SendMessageRight allows sending messages to a database by a set of names.

Note: commands are messages - so permission of sending commands is same as for messages.
type "sendMessage"right type
database string a specific database: 'test_db', multiple databases by prefix: 'test_*', all databases: '*'
names string[] a specific message: 'std.Echo', multiple messages by prefix: 'std.*', all messages: '*'

SubscribeMessageRight class extends Right

SubscribeMessageRight allows subscribing messages send to a database.

Note: commands are messages - so permission of subscribing commands is same as for messages.
type "subscribeMessage"right type
database string a specific database: 'test_db', multiple databases by prefix: 'test_*', all databases: '*'
names string[] a specific message: 'std.Echo', multiple messages by prefix: 'std.*', all messages: '*'

OperationRight class extends Right

OperationRight grant database access for the given containers based on a set of operations.
E.g. create, read, upsert, delete, query or aggregate (count)
It also allows subscribing database changes by subscribeChanges
type "operation"right type
database string a specific database: 'test_db', multiple databases by prefix: 'test_*', all databases: '*'
containers ContainerAccess[] grant execution of operations and subscriptions on listed containers
